Panel System

How widgets live on the infinite canvas

Each panel is:

  • A positioned div on the canvas — position: absolute, panned/zoomed with the viewport
  • Draggable via header bar — click + drag to reposition
  • Resizable via corner/edge handles — drag to resize
  • Lazy-mounted — React component mounts only when panel is "active"

Lazy Activation System

Canvas state: [panX, panY, zoom, activePanelId]
┌─ Panel visible on screen? ──→ YES → Widget.tsx mounts, initializes
│ connection to proxy
│
└─ Panel off-screen / minified? → Component unmounts →
   connection closes → zero resources

If a panel is scrolled off-canvas or zoomed too far out to be useful, it unmounts entirely. The MCP proxy also closes idle DB connections after 60s of no requests.

Panel State Machine

Closed → Placed (position set) → Visible (on screen, mounted) →
   Active (user interacting) → Idle (60s timeout) →
   Off-screen (unmounted) → Closed
       ↓
   Minimized (title bar only, no content)

Panel registry

A central registry defines available panel types. The user opens a panel from a toolbar/menu → it appears on the canvas at a default position. Position/size persisted in localStorage or project-instructions DB.
