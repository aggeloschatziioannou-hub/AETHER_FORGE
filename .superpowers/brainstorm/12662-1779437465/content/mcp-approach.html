How should the DB panel connect to databases?

The MCP servers run inside opencode — the web app can't talk to them directly

A

Direct SQLite via Next.js API

API route uses better-sqlite3 to read/write .db files. Simple, fast, no extra infra. Start with project-instructions.db, extend to any SQLite file later.

Pros

  • Simple to build
  • Zero extra infra
  • Fast queries

Cons

  • SQLite only
  • Need to configure allowed DB paths
B

MCP Proxy Server

Standalone backend that runs MCP server connections and exposes REST/WS to the frontend. Each panel connects through the proxy. Works with any MCP database server.

Pros

  • Any DB via MCP
  • True to vision C
  • Reusable for future panels

Cons

  • More infra to build
  • Extra process running
